## 三、架构说明（回应用户疑问）

### "后端配置图床"是什么意思？

项目分为两个独立应用：

| 应用 | 路径 | 用途 | 是否有图床字段 |
|------|------|------|-------------|
| **前端博客** | `XHBlogs/` | 访客看到的博客网站 | 无 picBedUrl/picBedToken |
| **后台管理** | `my-blog-manager/` | 博主发文章、传图片的后台 | 有 picBedUrl/picBedToken |

**图床上传逻辑**：
- 访客只看博客 → 不需要上传图片 → 不需要图床配置
- 博主在后台写 Markdown → 需要上传封面/插图 → 后台调用 Lsky Pro API 上传 → 返回图片 URL 嵌入文章

因此图床配置**只需在后台管理**（`my-blog-manager/siteConfig.ts`），前端博客不需要。
