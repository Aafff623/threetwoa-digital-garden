# 同事.skill｜炼化同事！打工人终极形态：变成一份 MD 文档～

> 本文核心：Colleague Skill 如何把同事工作痕迹炼成 AI 数字分身，以及它对职场就业和法律伦理的冲击。


### 核心主题

本期围绕开源项目 **Colleague Skill / 同事技能** 展开，讨论它如何把一个同事在工作中的聊天记录、文档、邮件、项目痕迹等信息，转化为一个可被 AI Agent 调用的"数字分身"。这个分身不仅能复用同事的专业知识和工作经验，还会尝试模拟其沟通风格、行为习惯与决策方式。

它的主要价值在于：把原本只存在于员工脑子里的经验、判断和踩坑记录，沉淀为团队可持续使用的数字资产；同时，它也引发了关于岗位替代、员工数据授权、人格权、个人信息保护以及 AI 行为责任归属等问题。

> 这期内容的重点不只是"AI 能不能模仿同事"，而是"人的工作痕迹能否被组织化、资产化、自动化调用"。真正敏感的地方也在这里。

### 核心概念

#### 1. Colleague Skill 是什么
- Colleague Skill 可以理解为一种"同事数字分身生成器"。
- 它会收集某个同事在工作中的多类数据，例如：
- - 聊天记录
- 工作文档
- 邮件内容
- 项目资料
- 本地文件
- 手动粘贴的文字材料
- 然后通过算法将这些信息拆分、整理，并生成符合 **Agent Skills 标准** 的 AI 技能包。
- 这个 AI 技能包具备两类能力：
- | 能力类型 | 含义 | 作用 |
|---|---|---|
| 硬技能 | 专业能力、业务知识、项目经验、工作方法 | 让 AI 能"干活" |
| 软特征 | 性格、沟通方式、表达习惯、行为模式、决策风格 | 让 AI 像"这个人" |
- 换句话说，它不是单纯做一个知识库问答机器人，而是试图复刻一个人的"工作能力 + 人格化工作风格"。

> 这里的关键不是"像不像本人聊天"，而是它能不能复现这个人在特定工作场景里的判断路径和交付方式。前者偏娱乐，后者才是生产力。

#### 2. 基本工作流程

> 配图说明：Colleague Skill 的双层架构——能力层（专业知识、项目经验、工作流程、决策依据）和人格层（表达风格、沟通习惯、性格标签、行为模式）共同构成数字分身。
- ```mermaid
flowchart TD
    A[收集同事工作痕迹] --> B[多源数据导入]
    B --> C[内容清洗与结构化]
    C --> D[拆分为硬技能与行为模式]
    D --> E[生成 Agent Skill 技能包]
    E --> F[部署到不同 AI Agent 平台]
    F --> G[持续增量更新]
    G --> H[版本管理与回滚]
```
- 整体逻辑是：
1. 从多个平台收集同事的工作数据；
2. 对数据进行整理、拆分、抽取；
3. 提炼出专业能力和人格化特征；
4. 打包成符合 Agent Skills 标准的技能；
5. 供团队或个人在不同 Agent 平台中调用；
6. 后续支持持续更新和版本回滚。

### 数据来源与接入方式
- Colleague Skill 的一个特点是数据来源比较宽，既支持自动同步，也支持手动上传。
- | 数据来源 | 接入方式 | 说明 |
|---|---|---|
| 飞书 | 自动拉取消息记录和文档 | 适合企业内部知识沉淀 |
| 钉钉 | 通过浏览器方式获取 | 自动化程度可能受平台限制 |
| Slack / 类似协作平台 | 通过 API 拉取 | 转写中提到"select"，语义上更可能指 Slack 一类工具 |
| 微信 | 导出聊天记录后上传 | 适合补充私域沟通内容 |
| 邮件 | 上传 EML 或 MBOX 格式文件 | 可保留正式沟通链路 |
| 本地文件 | 支持 PDF、图片、Markdown 等 | 可纳入项目资料、说明文档 |
| 手动文本 | 复制粘贴输入 | 适合补充零散经验和背景说明 |
> 多源数据接入越强，复刻效果越像；但同时，隐私风险、授权边界和数据污染问题也会越明显。

### 技术实现亮点


#### 1. 遵循 Agent Skills 开放标准
- Colleague Skill 生成的不是封闭格式，而是符合 **Agent Skills** 标准的 AI 技能包。
- 这意味着它具备一定的平台迁移能力，可以在不同 AI Agent 环境中使用，不必被某一个具体产品绑定。
- 主要好处包括：
- - 技能结构更清晰；
- 方便管理、升级和复用；
- 可迁移到不同 Agent 平台；
- 更容易形成团队内部的技能资产库。

#### 2. 支持增量更新与版本回滚
- 它不是一次性生成后就固定不变，而是可以根据新的数据持续更新。
- 这对于真实工作场景很重要，因为：
- - 项目背景会变；
- 团队成员职责会变；
- 同事的知识和判断也会更新；
- 旧版本可能存在错误，需要回退。
- 因此，增量更新和版本回滚可以让这个"数字同事"更接近真实工作中的动态状态。

#### 3. 多源数据融合
- Colleague Skill 不只依赖一种材料，而是把聊天、文档、邮件、本地资料等多种数据融合起来。
- 不同数据对应不同价值：
- | 数据类型 | 主要价值 |
|---|---|
| 聊天记录 | 反映沟通风格、即时判断、协作习惯 |
| 文档 | 反映系统化知识、方案设计、项目沉淀 |
| 邮件 | 反映正式表达、决策链路、责任边界 |
| PDF / 图片 / Markdown | 补充项目材料和知识背景 |
| 手动文本 | 补充隐性经验、上下文和个人说明 |
> 如果只喂文档，它更像知识库；如果加入大量聊天记录，它才会开始表现出"这个人说话和思考的味道"。

#### 4. 双层设计：能力层 + 人格层
- 该项目的核心设计可以理解为双层结构：
- ```mermaid
flowchart TD
    A[同事原始数据] --> B[能力层抽取]
    A --> C[人格层抽取]
-     B --> B1[专业知识]
    B --> B2[项目经验]
    B --> B3[工作流程]
    B --> B4[决策依据]
-     C --> C1[表达风格]
    C --> C2[沟通习惯]
    C --> C3[性格标签]
    C --> C4[行为模式]
-     B1 --> D[数字分身]
    B2 --> D
    B3 --> D
    B4 --> D
    C1 --> D
    C2 --> D
    C3 --> D
    C4 --> D
```

这也是它区别于普通知识库的地方：
普通知识库主要解决"知道什么"；Colleague Skill 还试图解决"这个人会怎么说、怎么判断、怎么推进"。

### 实际应用价值


#### 1. 知识传承：把员工经验变成团队资产
- Colleague Skill 最大的价值之一，是把原本只存在于员工大脑里的经验、习惯、判断和踩坑记录，转化为团队可以持续调用的数字资产。
- 典型场景包括：
- - 员工离职后，项目细节不再完全丢失；
- 新人可以快速了解项目背景；
- 团队可以复盘历史决策逻辑；
- 关键岗位的隐性经验可以被保留下来；
- 因人员流动导致的信息断层会减少。
- 尤其是很多团队真正的核心知识并不在正式文档里，而是散落在聊天记录、口头约定和个人经验中。Colleague Skill 的作用就是把这些"隐性知识"重新结构化。
> 很多公司缺的不是文档，而是"知道为什么当时这么做的人"。这类工具真正想保存的，是决策背后的上下文。

#### 2. 新人 onboarding：快速理解项目和团队
- 新人加入团队时，通常会遇到几个问题：
- - 不知道项目历史背景；
- 不知道关键决策为什么这么定；
- 不熟悉团队沟通风格；
- 不知道哪些坑已经被踩过；
- 不清楚找谁问什么问题。
- 如果有某位资深同事的 AI 分身，新人可以通过交互式问答快速了解：
- - 项目来龙去脉；
- 重要文档在哪里；
- 历史方案取舍；
- 常见问题处理方式；
- 团队内部的默认协作规则。
- 这比单纯阅读一堆文档更接近真实工作中的"带教"。

#### 3. 团队协作：让缺席的人也能提供部分支持
- 在团队协作中，Colleague Skill 可以用于：
- - 模拟某位同事的专业回复；
- 在对方不在线时提供初步判断；
- 帮助跨部门团队理解对方沟通方式；
- 让项目在成员缺席时继续推进；
- 沉淀团队默契和关键上下文。
- 比如某个产品经理、架构师、销售负责人临时不在，但他的 AI 分身可以先回答一些背景问题，或者提供接近其风格的反馈。
- 当然，这不等于完全替代本人，而是用于降低沟通等待成本。
> 比较现实的定位是"异步协作增强器"，不是"真人完全替身"。它适合处理背景解释、经验复用和初步判断，不适合承担最终责任。

#### 4. 个人生活场景：私人助手与情感陪伴
- 节目中也提到，这类技术不一定只用于企业，还可能进入个人生活场景。
- 例如：
- - 把自己的沟通方式沉淀成个人数字分身；
- 把朋友、家人、前任的聊天记录整理成 AI 角色；
- 用于陪伴、回忆或模拟对话；
- 形成一种持续进化的私人助手。
- 这类用法技术门槛较低，但伦理边界更复杂。
- 尤其是未经他人同意，使用聊天记录复刻对方，很容易触及隐私、人格权和情感伤害问题。
> 工作场景里，问题主要是授权和责任；生活场景里，问题更微妙，涉及情感操控、关系边界和逝者/前任/亲友的数字人格使用权。

### 对就业的影响


#### 1. 标准化白领岗位最容易受到冲击
- 节目中提到，已有一些公司使用 AI 数字分身去替代或压缩部分岗位，尤其是：
- - 销售；
- 客服；
- 内容创作；
- 标准化运营；
- 流程化支持岗位。

- 这些工作的共同特点是：
- | 特征 | 替代风险 |
|---|---|
| 流程标准化 | 高 |
| 输出模板化 | 高 |
| 决策空间有限 | 高 |
| 依赖已有话术和资料 | 高 |
| 可通过历史数据训练 | 高 |
- 因此，这些岗位可能会出现：
- - 人员精简；
- 招聘减少；
- 新人机会变少；
- 中年职场人转岗压力增大；
- 岗位门槛提高；
- 晋升空间压缩。

#### 2. 工作不会简单消失，但能力要求会重排
- Colleague Skill 这类工具并不意味着所有打工人都会马上失业，但会改变岗位价值排序。
- 更容易被压缩的是：
- - 只会执行固定流程的人；
- 只会复述已有信息的人；
- 输出高度模板化的人；
- 无法处理复杂判断的人；
- 没有跨领域整合能力的人。
- 更不容易被替代的是：
- - 能定义问题的人；
- 能做复杂判断的人；
- 能承担责任的人；
- 能处理模糊场景的人；
- 能建立信任关系的人；
- 能创造新方法、新业务、新结构的人。
> AI 最先吃掉的不是"努力"，而是"可被标准化描述的努力"。所以真正要提升的不是忙碌程度，而是判断力、创造力和责任能力。

### 法律与伦理风险


#### 1. 员工数据不能被企业随意使用
- 如果企业要收集员工的语音、图像、沟通风格、聊天记录、文档内容等数据，用于训练 AI 复刻人，必须获得员工明确同意。
- 涉及的权益包括：
- - 人格权；
- 肖像权；
- 声音权益；
- 个人信息权益；
- 通信隐私；
- 劳动权益；
- 数据使用知情权。
- 企业不能因为员工在公司工作，就默认拥有其全部工作痕迹的任意使用权。
-

#### 2. 未经授权训练 AI 可能带来法律责任
- 节目中强调，如果企业未经员工同意就用其数据训练 AI，可能涉及：
- - 民事赔偿；
- 个人信息侵权；
- 劳动争议；
- 数据合规问题；
- 情节严重时甚至可能产生刑事责任风险。
- 尤其当 AI 分身对外执行任务时，责任会更加复杂。
- 例如：
- - AI 代表某员工回复客户，责任算谁的？
- AI 模拟某员工签合同，合同效力如何认定？
- AI 给出错误业务判断，损失由谁承担？
- AI 使用的数据泄露，企业如何赔偿？
- 员工离职后，企业还能否继续使用其数字分身？
- 这些问题都需要明确制度约束，而不能靠"技术能做到"来替代法律授权。
> AI 分身最麻烦的地方在于它模糊了"谁在说话、谁在决策、谁来负责"。企业如果不提前划清边界，后续风险会非常大。

### 应对建议


#### 对普通打工人
- 普通员工需要重点关注自己的数据权利和职业不可替代性。
- 建议：
1. **了解公司如何使用个人信息**  
   包括聊天记录、邮件、文档、语音、会议纪要等是否被用于训练 AI。
- 2. **关注授权文件和制度说明**  
   不要随意签署过于宽泛的数据授权条款。
- 3. **保存相关证据**  
   如果发现公司未经同意使用个人数据，应保留通知、合同、系统截图、邮件记录等。
- 4. **提升不可替代能力**  
   尽量向复杂判断、跨部门协调、创造性工作、责任型岗位发展。
- 5. **建立个人知识资产**
  不只是被公司"炼化"，也要主动沉淀自己的方法论、作品集和行业理解。

#### 对企业
- 企业如果要使用 Colleague Skill 这类工具，不能只考虑效率，也必须建立配套治理机制。
- 建议：
1. **建立员工数据授权机制**  
   明确收集哪些数据、用于什么目的、保存多久、谁能访问。
- 2. **明确 AI 分身的使用边界**  
   哪些场景可以用，哪些场景不能用，是否允许对外代表员工发言。
- 3. **明确责任归属**  
   AI 做出的建议、回复、合同动作、业务判断，都需要有责任主体。
- 4. **提供退出和删除机制**  
   员工离职或撤回授权后，企业应明确如何处理相关数据和模型。
- 5. **加强员工培训与转岗支持**  
   技术进步不应只转化为裁员工具，也应帮助员工升级能力。
- 6. **做好数据安全防护**  
   防止员工画像、沟通记录、业务机密等敏感信息泄露。
> 企业最稳妥的做法，不是偷偷把员工"炼成 AI"，而是把它作为知识管理和协作增强工具，并给员工明确的知情权、选择权和收益边界。
