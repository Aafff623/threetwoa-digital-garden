# Pi Agent：比 Claude Code 和 Codex 更适合普通人的 AI 工具

> 本文结构：核心结论 → 概念拆解 → 操作步骤 → 综合案例 → 提示词模板。建议按需跳转。


### 一句话总结

这期视频的核心不是单纯教你安装一个工具，而是在讲一种更适合普通人的 Agent 使用方法：**用一个极简的本地 Agent 作为底座，再通过 Skill 按需扩展能力，把搜索、读文档、写稿、配音、生图、做视频等日常任务串成完整工作流。**

> 视频真正想强调的是"工作流效率"，而不是"某个工具有多强"。Pi Agent 的价值在于它不像 Coding Agent 那样默认围绕写代码设计，而是更适合把普通办公、调研、内容生产这类任务变成可交付的文件。

### 核心主题：为什么普通人应该关注本地 Agent
- 视频认为，**本地 Agent 是当前 AI 发展中最重要的方向之一**，也是普通人跟上 AI 时代最直接的方式。
- 如果用户目前只是每天和聊天机器人问答，而没有真正接触过本地 Agent，就会遇到几个问题：
- - AI 只能在对话框里回答，无法真正接管本地任务。
- 搜索、读文件、写文稿、生成音频、生成图片、制作视频等步骤仍然割裂。
- 普通人面对 Agent、Skill、MCP、模型配置等概念时容易无从下手。
- 视频给出的入门框架很简单：

  ```text
  Agent + Skill = 普通人可用的 AI 工作流
  ```

- 也就是说，Agent 负责理解任务、拆解任务、调用工具；Skill 则像一份操作说明书，让 Agent 学会某一类具体能力。

### 核心概念


#### 1. Coding Agent 与日常任务 Agent 的区别
- 视频把当前主流 Agent 大致分成两类：
- | 类型 | 代表工具 | 主要目标 | 最终产出 | 更适合的人群 |
|---|---|---|---|---|
| Coding Agent | Claude Code、Codex、OpenCode | 写代码、改项目、跑测试、处理 Git | 项目文件、代码变更 | 程序员、开发者 |
| 日常任务 Agent | Pi Agent | 搜资料、读文件、写汇报、做内容、串工作流 | 文档、网页、音频、图片、视频、调研报告 | 普通办公用户、内容创作者、研究人员 |
- Claude Code、Codex、OpenCode 这类工具的设计重心是"交付代码"。它们通常内置了代码索引、测试运行、Git 操作、编码规范等一整套工程能力。
- 但普通人的日常任务往往不是写代码，而是：
  - 搜索资料；
  - 阅读 PDF、Word、PPT、Excel；
  - 整理表格；
  - 写汇报；
  - 做 PPT 或演示；
  - 生成语音；
  - 生成图片；
  - 制作短视频或动画演示。
- 所以，直接拿 Coding Agent 处理这些任务，虽然也能做，但并不一定是最合适的。

> 这里的关键不是"Pi Agent 比 Claude Code 强"，而是"它们服务的任务类型不同"。Claude Code 适合开发工程，Pi Agent 更像一个可组装的个人工作台。

#### 2. Pi Agent 的设计理念：极简底座 + 按需加 Skill
- Pi Agent 与其他 Agent 最大的不同，是它默认非常克制。
- 它的底座只保留四个基础能力：
1. 读文件；
2. 写文件；
3. 改文件；
4. 跑命令。
- 除此之外，其他能力都不预装。
- 它的设计思路是：**不要默认塞进一大堆通用能力，而是让用户自己搭建一个适合自己的 Agent。**
- 不同用户可以按需安装不同 Skill：
- | 用户需求 | 可以安装的 Skill |
|---|---|
| 做研究 | 搜索 Skill、PDF Skill |
| 办公处理 | Word、PPT、Excel Skill |
| 语音生成 | Edge TTS Skill |
| 图片生成 | GPT Image 相关 Skill |
| 视频/演示生成 | Hyper Frames Skill |
- 因此，每个人最终用到的 Pi Agent 都不一样。
- 视频引用官网口号表达这一理念：

  ```text
  世界上有很多不同类型的 Agent，但 Pi Agent 是你自己的 Agent。
  ```

#### 3. Skill 是什么
- 视频中反复提到的 Skill，可以理解为：
- ```text
  一份写给 Agent 的说明书 / 操作手册
  ```
- Agent 读完 Skill 之后，就知道某类任务该怎么处理。
- 比如：
  - 安装搜索 Skill 后，Agent 才知道如何联网查资料；
  - 安装 PDF Skill 后，Agent 才知道如何解析 PDF；
  - 安装 Edge TTS Skill 后，Agent 才能生成语音；
  - 安装 Hyper Frames Skill 后，Agent 才能做动画网页或视频演示。
- Skill 的作用不是单独完成任务，而是赋予 Agent 某种可调用能力。

#### 4. Pi Agent 为什么更轻、更快、更省 Token
- 视频用系统提示词长度做了一个对比：
- | 工具 | 系统提示词大致长度 |
|---|---|
| Claude Code | 约 2 万 token |
| Codex / OpenCode | 约 1.5 万 token |
| Pi Agent | 不到 1500 token |
- 这会带来三个直接影响：


##### 第一，速度更快
- 上下文更短，模型需要处理的信息更少，因此响应和推理通常更快。


##### 第二，更省 Token
- 视频作者的体感是，完成同样规模的日常任务，Pi Agent 的 Token 消耗大约只有 Claude Code 的三分之一，甚至更低。


##### 第三，模型更"听话"
- 模型的注意力有限。如果系统提示词太长，模型的注意力会被大量预设规则占据。
- 对于日常任务来说，Pi Agent 没有一万多字的编程规则干扰，因此更容易专注于当前任务本身。
> 这里的判断带有作者个人使用体感，但逻辑是成立的：系统提示词越短，任务上下文中留给用户目标和实际资料的空间越大。对于非编程任务，这种简洁性确实有优势。

### Pi Agent 的整体工作流逻辑

> 配图说明：Pi Agent 工作流的核心是从"用户目标"出发，Agent 智能拆解后按需调用各类 Skill，最终交付文档、网页、音频、图片或演示项目等可交付物。
- ```mermaid
flowchart TD
    A[用户提出目标] --> B[Pi Agent 理解并拆解任务]
    B --> C{需要什么能力}
    C --> D[搜索 Skill：联网查资料]
    C --> E[PDF / Office Skill：读取本地文件]
    C --> F[Edge TTS：生成语音]
    C --> G[GPT Image：生成图片]
    C --> H[Hyper Frames：生成动画演示]
    D --> I[整理资料与结构化输出]
    E --> I
    F --> J[生成音频文件]
    G --> K[生成图片素材]
    H --> L[生成 HTML / 演示页面 / 视频素材]
    I --> M[交付文档、网页、音频、图片或演示项目]
    J --> M
    K --> M
    L --> M
```

## 核心步骤

### Step 1：安装基础环境 Node.js
- 安装 Pi Agent 前，需要先安装 Node.js。
- 视频强调，Node.js 几乎是安装各类 Agent 时经常会依赖的基础环境。
- 操作方式：
1. 前往 Node.js 官网；
2. 根据自己的系统选择安装包；
3. 下载后双击安装；
4. 一路点击"下一步"完成安装；
5. 打开终端检查后续操作。
- 如果此前已经安装过 Node.js，可以跳过这一步。

### Step 2：可选配置 npm 国内镜像加速
- 视频中提到，如果在国内网络环境下安装较慢，可以配置 npm 国内镜像。
- 作者使用的是腾讯的镜像加速。
> 转写文本中没有给出完整命令，因此这里不补写具体命令，避免误导。实际使用时可以搜索"npm 国内镜像加速"或按项目文档配置。

### Step 3：启动 Pi Agent Web 应用
- Pi Agent 默认运行在命令行中，界面对普通用户不算友好。
- 作者为它封装了一层网页界面，做成了一个更容易使用的 Pi Agent Web 应用。后续演示也基于这个 Web 版本。
- 启动方式：
- - Windows：打开 PowerShell；
- macOS：打开终端；
- 输入作者提供的启动命令；
- 首次运行会下载项目文件，因此速度较慢；
- 后续更新会从缓存加载，速度会快很多；
- 安装完成后会自动弹出网页应用。
> 转写里没有保留具体启动命令，笔记中不强行补全。真正操作时应以作者项目 README 或视频简介区命令为准。

### Step 4：认识 Pi Agent Web 界面
- Pi Agent Web 大致包含几个区域：
- | 区域 | 功能 |
|---|---|
| 左侧文件浏览器 | 选择或查看当前工作目录中的文件 |
| 中间对话框 | 输入任务指令、上传图片、与 Agent 对话 |
| 模型设置区 | 添加模型服务商、配置模型参数 |
| Skills 设置区 | 添加、安装、管理 Skill |
| 文件预览区 | 查看 Agent 生成的 Markdown、HTML 等文件 |
- 安装完成后，即使没有额外添加 Skill，Pi Agent 也已经能做一些本地任务：
  - 读取桌面文件；
  - 整理文件夹；
  - 批量修改文件名；
  - 执行命令；
  - 运行脚本；
  - 处理通过本地文件和命令能完成的任务。

### Step 5：配置模型服务商与模型
- Agent 本身没有模型能力，需要先配置模型接口。
- 视频中的配置流程：
1. 点击"模型"；
2. 点击"添加模型服务商"；
3. 从官方供应商列表中选择；
4. 如果列表里没有，就选择 `custom` 自定义接口；
5. 填写：
   - Base URL；
   - API Key；
6. 保存配置；
7. 添加具体模型；
8. 填写模型名称；
9. 根据模型能力勾选参数。
- 视频中演示使用了一个能力较强、上下文较大的模型，并配置了：
- | 参数 | 视频中的配置思路 |
|---|---|
| 推理模式 | 开启 |
| 图片输入 | 允许 |
| 思考强度 | 选择较强模式 |
| 上下文窗口 | 约 20 万 |
| 最大输出 | 约 6.4 万 token |
- 配置完成后，作者用"你好"和"看一下我的桌面上有什么"测试模型是否连通、是否能读取本地目录。

## Skill 安装与能力扩展
- 视频的核心实践部分，是逐个给 Pi Agent 安装 Skill，并立即用小任务验证能力。


### Skill 安装位置：Global 与 Project
- 安装 Skill 时有两个选项：
- | 选项 | 含义 | 适用场景 |
|---|---|---|
| Global | 全局安装，所有项目都可以使用 | 常用通用能力 |
| Project | 只在当前项目中可用 | 某个项目专用能力 |
- 作者通常默认选择 Global，因为搜索、读文件、语音、生图、视频等能力会在很多项目中反复使用。

### Step 6：安装搜索 Skill

#### 推荐搜索服务
- 视频推荐了两个搜索服务：
- | 搜索服务 | 优点 | 注意点 |
|---|---|---|
| Tavily Search | 注册简单，免费账号每月约 1000 次搜索额度，适合新手 | 搜索质量够用 |
| Brave Search | 搜索结果更好 | 需要绑定信用卡，配置稍麻烦；免费额度同样约 1000 次 |


#### Tavily Search 配置流程
1. 在 Skills 中搜索 `Tavily Search`；
2. 点击安装；
3. 前往 Tavily Search 官网注册免费账号；
4. 获取 API Key；
5. 回到 Pi Agent；
6. 直接告诉 Agent：帮我设置 Tavily Search 用的 Key；
7. 粘贴 Key；
8. Agent 会自动写入配置。


#### 搜索任务演示
- 作者给出的任务是：
- ```text
  搜一下这周最重要的五条 AI 新闻，按重要程度排序。
  每条说明发生了什么、为什么重要、适不适合做短视频。
  保存到一个 news.md 文件。
  ```
- Pi Agent 的处理过程不是简单返回一堆链接，而是会：
1. 调用搜索 Skill；
2. 搜索相关新闻；
3. 评价搜索结果；
4. 去重；
5. 排序；
6. 判断重要性；
7. 判断是否适合做短视频；
8. 生成 `news.md` 文件；
9. 保存到当前目录。
- 这说明搜索 Skill 不只是"联网"，而是可以配合 Agent 完成资料筛选和结构化输出。

### Step 7：安装 PDF / 文件读取 Skill

#### PDF Skill
- 视频推荐的是 OpenAI 发布的 PDF Skill。
- 它的能力分两类：
- | PDF 类型 | 处理方式 |
|---|---|
| 文字版 PDF | 提取文字后阅读 |
| 扫描版 PDF | 需要模型开启图像识别能力，让模型看 PDF 图像内容 |


#### PDF 阅读演示
- 作者用 DeepSeek V4 技术报告测试：
1. 将 PDF 拖入工作目录；
2. 刷新文件浏览器；
3. 在对话中 `@` 这份 PDF；
4. 输入任务：
- ```text
  读取这个 PDF，总结里面的核心信息。
  ```
- Pi Agent 会自动调用 PDF Skill，读取十几页报告，并整理出结构化总结。
- 关键价值在于：
- - 不需要手动复制 PDF 内容；
- 不需要用户自己提取文字；
- 只要把文件交给 Agent，它会自行处理。


#### Office 文件能力
- 视频还提到，类似能力可以扩展到 Office 文件：
- | 文件类型 | 对应能力 |
|---|---|
| Word | 阅读和整理文档 |
| PPT | 拆解演示文稿 |
| Excel | 处理表格 |
- 其中 PPT 的门道最多，视频中没有展开，作者表示可以后续单独讲。
> 这部分其实非常适合普通人。相比"让 AI 写代码"，大多数办公用户更需要的是让 AI 直接读懂手头的 PDF、Word、PPT 和 Excel。

### Step 8：安装 Edge TTS，让 Agent 生成语音
- 第三类能力是让 Pi Agent "开口说话"。
- 作者安装的是 Edge TTS Skill。
- 特点：
- - 不需要额外账号；
- 安装后即可使用；
- 可以把文稿转成语音文件。


#### 任务演示：新闻简报转口播音频与网页
- 作者基于之前生成的 `news.md` 文件，给出任务：
- ```text
  把这份文件改成一分钟的口播稿，转成语音。
  做一个 HTML 界面，里面包含文稿和音频，排版简单直观。
  然后帮我打开这个网页。
  ```
- Pi Agent 的执行过程：
1. 读取 `news.md`；
2. 改写成一分钟口播稿；
3. 调用 Edge TTS 生成 MP3 音频；
4. 写一个 HTML 页面；
5. 把文稿和音频嵌入网页；
6. 自动打开网页。
- 生成结果包括：
- - 一份口播稿；
- 一个 MP3 音频文件；
- 一个 HTML 页面；
- 可直接播放音频的网页界面。
- 这一步已经形成了一个小型工作流：

  ```mermaid
  flowchart TD
      A[news.md 新闻简报] --> B[Pi Agent 改写口播稿]
      B --> C[Edge TTS 生成音频]
      C --> D[Pi Agent 编写 HTML 页面]
      B --> D
      D --> E[打开可分享的网页结果]
  ```

> 这是 Pi Agent 和普通聊天机器人的关键区别。普通聊天机器人通常把结果留在对话框里，而 Pi Agent 能把结果落到本地文件、网页、音频这些可交付物上。

### Step 9：安装图片生成 Skill
- 第四类能力是生成图片。
- 视频中安装的是 GPT Image 相关 Skill，作者称其调用 Codex 中的 GPT Image 生图能力。
- 适用条件：
- - 需要订阅 GPT Plus 或 Pro；
- 然后可以把生图能力交给 Pi Agent 调用。


#### 生图任务演示
- 作者仍然基于前面生成的 AI 新闻文件，给出任务：
- ```text
  根据这个文件的第一条 AI 新闻，
  用 GPT Image 生成一张适合短视频封面的图片。
  风格是简洁的 3D 科技感图片。
  ```
- 第一条新闻主题是 OpenAI 的推理模型在数学难题上找到新构造。
- 生成图像结果符合要求：
- - 有科技感；
- 适合短视频封面；
- 主题贴合 AI 与数学突破；
- 风格简洁偏 3D。


#### Windows 报错处理
- 作者特别提醒：
- - Mac 上通常可以直接使用；
- Windows 上如果遇到脚本报错，不必紧张；
- 可以直接让 Pi Agent 检查 Skill 脚本哪里有 bug；
- 让它尝试修复后再运行。
> 这也是本地 Agent 的一个实际优势：不仅能调用工具，还能读脚本、改脚本、修复运行错误。遇到环境问题时，可以把 Agent 当成"会动手的排错助手"。

### Step 10：安装 Hyper Frames，让 Agent 做动画演示 / 视频
- 第五类能力是制作视频或动画演示。
- 作者安装的是 `Hyper Frames` Skill。
- 它适合做：
- - 讲解类视频；
- 产品介绍；
- 科普动画；
- 过程演示；
- 带字幕和动画的演讲页面。


#### Hyper Frames 的工作原理
- 它不是直接让 AI 生成视频，而是采用更稳定的流程：
1. 先让 Agent 写一个带动画的 HTML 网页；
2. HTML 由代码组成，结构稳定；
3. 用户可以预览、编辑；
4. 再逐帧渲染成完整视频。
- 这个思路的优势是：
- | 优势 | 说明 |
|---|---|
| 稳定 | HTML 可控，不像纯生成视频那样随机 |
| 可编辑 | 页面、动画、字幕、结构都能改 |
| 可预览 | 生成后能先打开查看 |
| 门槛低 | 用户不需要理解 HTML 代码，只要描述想要的效果 |


#### 小 Demo 演示
- 作者给出的任务是：
- ```text
  用 Hyper Frames 做一个 20 秒的动画，
  解释什么是 Agent。
  只生成 HTML，不需要渲染视频。
  ```
- Pi Agent 最终生成一个动画 HTML 页面，包含：
- - 标题；
- 转场；
- 图形动画；
- 字幕节奏；
- 基础讲解结构。

### 调研内容核心整理

- 根据视频展示，Pi Agent 生成的演讲大致包含以下信息：


#### 1. GB200 NVL72 的基础规格
- 英伟达官方资料显示，NVL72 是一个液冷机柜系统，内部连接：
- - 36 个 Grace CPU；
- 01:12 个 Blackwell GPU；
- 组成一个 72 GPU 的 NVLink 域；
- 整柜 NVLink 通信带宽约 130 TB/s；
- HBM3E 显存总量约 13.4 TB。


#### 2. 价格区间
- 公开资料中没有英伟达官方标价。
- 视频中提到的估算来源包括：
- | 来源 | 估算信息 |
|---|---|
| Tom's Hardware 引用业内来源 | 约 280 万至 340 万美元 |
| HSBC 2024 年估计 | 约 300 万美元 |
- 因此，案例中以 300 万美元作为中位数进行拆解。


#### 3. 最大成本：GB200 Superchip
- 视频中提到，HSBC 估计每颗 GB200 Superchip 约 6 万至 7 万美元。
- 如果一柜包含 36 颗，则合计约：
- ```text
  36 × 6万～7万美元 = 216万～252万美元
  ```
- 这大约占整柜价格的七成到八成。
- GB200 Superchip 的主要价值来自：
- - 两颗 Blackwell GPU；
- Grace CPU；
- HBM；
- 先进封装。
- 视频中还提到 EPIC AI 的估算：单颗 GB200 制造成本约 6400 美元，其中：
- | 成本项 | 占比 / 特点 |
|---|---|
| HBM | 接近一半 |
| 先进封装 | 约 1100 美元 |
| 逻辑硅片 | 低于 15% |
> 这里要注意区分"制造成本估算"和"最终售价"。芯片最终售价会包含研发、供应稀缺、生态绑定、利润、质保、整机集成等因素，不能简单用制造成本倒推售价。

#### 4. 第二层成本：互联、网络与系统部件
- NVL72 机柜中还包括：
- - 9 个 NVLink Switch Tray；
- ConnectX 网卡；
- BlueField DPU；
- 管理网络；
- 本地存储；
- 其他系统部件。
- 这些部分没有非常明确的公开逐项价格。
- 按 300 万美元总价扣除 Superchip 后，互联网络与其他系统件合计大约是几十万美元量级。


#### 5. 第三层成本：液冷、电力与工程集成
- NVL72 的工程要求非常高：
- - 功耗约 120 kW；
- 重量约 1.36 吨；
- 需要液冷；
- 需要高规格供电；
- 需要机柜结构设计；
- 需要测试和整柜集成。
- 视频中引用 Tom's Hardware / Morgan Stanley 相关数据，提到 GB300 NVL72 冷却部件约 4.986 万美元。
- 但作者强调，这个数字只代表冷却部件本身的量级，不包含：
- - 机房侧 CDU；
- 配电；
- 运维；
- 更完整的数据中心侧工程成本。


#### 6. 结论：300 万美元主要花在哪里
- 视频中的总结可以归纳为：
- | 成本层级 | 主要内容 | 确定性 |
|---|---|---|
| 第一层 | GPU、HBM、先进封装、NVLink 生态 | 相对确定，是最大头 |
| 第二层 | 网络互联、Switch、DPU、网卡、系统部件 | 部分公开，具体拆分不透明 |
| 第三层 | 液冷、电力、结构、测试、集成 | 有估算，但完整成本依场景变化 |
| 外层溢价 | 供应稀缺、软件栈、质保、整柜集成溢价 | 高度依赖合同与采购量 |
- 最不确定的两项是：
1. 实际成交价；
2. 非芯片部分的精确拆分。
- 它们会随着客户采购量、合同条款、交付周期和供应情况变化。

## 视频中的关键方法论

### 1. 不要只把 Agent 当聊天机器人
- 普通聊天机器人主要输出文本回答。
- Pi Agent 这类本地 Agent 更重要的是能生成和操作本地交付物：
- - Markdown 文档；
- HTML 页面；
- 音频；
- 图片；
- 演示项目；
- 视频素材；
- 结构化文件夹。
- 这意味着 AI 不只是"告诉你怎么做"，而是可以直接"帮你把东西做出来"。

### 2. Skill 的意义是把能力模块化
- 视频反复展示的逻辑是：
- ```text
  装搜索，它就能联网；
  装 PDF / Office，它就能读资料；
  装 TTS，它就能说话；
  装 GPT Image，它就能生成图片；
  装 Hyper Frames，它就能做动画演示。
  ```
- 每安装一个 Skill，Pi Agent 就获得一类新能力。
- 当这些 Skill 被串起来后，就形成了真正的工作流。

### 3. Coding Agent 提升开发效率，Pi Agent 提升工作流效率
- 视频最后对两类 Agent 做了一个非常明确的区分：
- | 工具类型 | 解决的问题 |
|---|---|
| Coding Agent | 开发效率 |
| Pi Agent | 工作流效率 |
- Coding Agent 的产出通常是代码。
- Pi Agent 的产出则是结果，代码只是中间手段。
- 比如生成一个调研演讲项目时，Pi Agent 可能会写 HTML、调用脚本、生成文件，但用户并不关心中间代码，用户关心的是最终交付：
- - 调研是否完整；
- 来源是否可靠；
- 演讲稿是否清楚；
- 配音是否生成；
- 页面是否能打开；
- 演示是否能直接预览。

> 这点很重要。普通人用 Agent，不一定要理解它写了什么代码，而是要学会描述目标、检查结果、补充约束。
