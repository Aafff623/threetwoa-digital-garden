# Claude Code 最实用的 15 个 Skills 推荐：从进化类到蒸馏类

> 本文结构：六大类别 → 每类2-5个Skill → 推荐安装顺序 → 总览表。建议按需跳转。


### 核心结论

这期内容围绕 Claude Code 的 Skills 生态做了一次实用向筛选：不是追求"装得多"，而是从高下载量、实际使用频率、踩坑经验出发，挑出 15 个更适合长期放进工作流里的 Skills。

整体可以分为六大类：

| 类别 | 解决的问题 | 推荐对象 |
|---|---|---|
| 进化类 | 让 AI 记住错误、主动执行任务 | 高频使用 Claude Code 的用户 |
| 开发必备 | GitHub 协作、工程规范、TDD、提交管理 | 开发者、团队项目维护者 |
| 信息获取 | 多源搜索、网页操作、文档总结 | 技术调研、资料整理、竞品监控 |
| 效率办公 | Google Workspace、Notion、Obsidian 自动化 | 知识管理和办公自动化用户 |
| 安全维护 | 检查 Skill 权限与风险、自动更新 | 所有安装 Skills 的用户 |
| 蒸馏类 | 将人、角色或知识体系转成 AI Skill | 团队协作、个人知识复刻、兴趣玩法 |

> 核心判断很明确：Skills 的价值不在于数量，而在于它是否真的进入你的日常工作流。装十个不用，不如装两个天天用。

### 工具清单


#### 一、进化类 Skills：让 AI 越用越聪明


##### 1. Self Improving Agent
- Self Improving Agent 的核心作用是让 Claude Code 具备"记性"。
- 它会记录两类信息：
- - 用户纠正 Claude 的地方；
- Claude 执行任务失败的地方。
- 当下次遇到类似任务时，它会先查阅之前的错误日志，再决定怎么执行。
- 未安装时，Claude 可能今天犯过的错，明天还会再犯；安装之后，它会逐渐积累你的偏好、项目习惯和常见失败案例。
- 适合场景：
- - 长期维护同一个项目；
- 经常纠正 AI 的代码风格、命名习惯、目录结构；
- 希望 AI 逐渐适应个人或团队规范；
- 想减少重复沟通成本。
> 这是典型的"越早装越值钱"的 Skill。它的收益不是一次性的，而是随着使用时间不断累积。

##### 2. Proactive Agent
- Proactive Agent 改变的是 Claude 默认的被动工作模式。
- 默认情况下，Claude 通常是：
- ```text
  你说一句 → 它做一句
  ```
- 安装 Proactive Agent 后，可以让 Claude 主动执行预设任务，例如：
- - 每天早上 9 点自动跑一次竞品监控；
- 每次提交代码后自动生成变更摘要；
- 定时检查某些页面或数据变化；
- 自动触发一组工作流。
- 适合场景：
- - 定时监控；
- 自动生成日报、周报；
- CI/CD 辅助说明；
- 项目状态周期性检查。
> 这个 Skill 的价值不只是"省一步操作"，而是把 AI 从聊天助手变成半自动执行器。

#### 二、开发必备 Skills：代码协作与工程规范


##### 3. GitHub Skill
- 这是 GitHub 开发者非常值得安装的 Skill。
- 安装后，Claude 可以直接在对话中操作 GitHub，例如：
- - 创建分支；
- 提交 Pull Request；
- 处理 Issue；
- 查询 CI 状态；
- 修改代码后自动写 PR 描述；
- 根据 Bug 描述完成修复并发起 PR。
- 典型使用方式：
- ```text
  帮我修一下这个 bug，然后提一个 PR。
  ```
- Claude 可以完成从代码修改到 PR 描述生成的一整套流程，减少在浏览器、IDE、GitHub 页面之间来回切换。
- 适合场景：
- - GitHub 项目维护；
- 开源项目协作；
- Issue 修复；
- PR 自动化；
- CI 状态追踪。
> 如果你的主要代码协作平台是 GitHub，这个基本属于刚需级 Skill。

##### 4. Super Powers
- Super Powers 是 Jesse Vincent 做的一套开发工作流 Skill，并且已经上架官方插件市场。
- 它的特点是"重流程、重规范"。
- 它不会让 Claude 一上来就直接写代码，而是要求先完成：
1. 需求梳理；
2. 设计确认；
3. 任务拆分；
4. 测试设计；
5. 再进入代码实现。
- 它还强制要求 TDD，也就是：
- ```text
  先写测试 → 再写实现
  ```
- 优点：
- - 降低 AI 写偏、跑题、误改的概率；
- 更适合复杂需求；
- 适合多人协作项目；
- 能提升需求到实现之间的可追踪性。
- 缺点：
- - 流程偏重；
- 小项目会显得繁琐；
- 不适合临时脚本或轻量需求。
- 适合场景：
- - 中大型项目；
- 团队协作；
- 需要严格工程规范的代码库；
- 对 TDD 有明确要求的项目。
> Super Powers 更像是给 Claude 套了一层工程经理和架构师的约束，让它不要急着"开写"。

##### 5. Wassa to93
- Wassa to93 走的是轻量工程规范路线。
- 如果觉得 Super Powers 太繁琐，可以选择它。
- 它主要集中管理一些工程中常用的规范，例如：
- - 代码风格约定；
- 提交信息格式；
- 文档模板；
- 常见项目结构规范；
- 开发过程中的基本约束。
- 它不会强制走完整的 TDD 流程，因此更适合个人开发者或者轻量项目。
- 对比关系：
- | Skill | 定位 | 适合对象 |
|---|---|---|
| Super Powers | 严格模式 | 团队项目、中大型项目 |
| Wassa to93 | 够用模式 | 个人项目、轻量开发 |
> 如果说 Super Powers 是"团队工程规范"，Wassa to93 更像是"个人开发者的实用约束包"。

#### 三、信息获取类 Skills：搜索、浏览与总结


##### 6. Multi Search Engine
- Multi Search Engine 解决的是搜索来源单一的问题。
- 普通情况下，Claude 可能只从一个搜索来源获取结果。但这个 Skill 可以同时搜索多个搜索引擎，然后聚合结果再回答。
- 适合场景：
- - 技术选型调研；
- 框架对比；
- 竞品研究；
- 问题排查；
- 查找不同来源的观点和资料。
- 它的优势在于，同一个问题，不同搜索引擎可能会给出不同角度的信息。多源结果拼起来，通常比单一来源更完整、更可靠。
> 做技术调研时，多源搜索比单点搜索更重要，因为不同搜索引擎的索引范围和排序逻辑并不一样。

##### 7. Agent Browser
- Agent Browser 让 Claude 不只是"搜索网页"，而是真的可以操作浏览器。
- 它底层使用的是 Playwright，Claude 可以控制真实浏览器完成：
- - 点击链接；
- 填写表单；
- 截图；
- 抓取页面内容；
- 监控页面变化；
- 执行网页上的交互操作。
- 案例包括：
- - 自动抢会议室；
- 监控竞品价格；
- 抓取动态页面内容；
- 自动填写重复表单。
- 需要注意的是，它操作的是真实浏览器，权限比较大。因此不要在不信任的网站上随便使用，也不建议让它处理敏感账号、支付页面或高权限后台。
> Agent Browser 很强，但也意味着风险更高。凡是能"替你点按钮"的工具，都应该谨慎授权。

##### 8. Summarize
- Summarize 是一个多格式总结工具。
- 支持内容包括：
- - 网页；
- PDF；
- 图片；
- 音频；
- 视频；
- 长文档。
- 常见用法：
- - 阅读长文章；
- 总结新框架文档；
- 提炼 PDF 报告；
- 快速了解几十页甚至上百页的资料；
- 先生成概要，再决定哪些章节值得细读。
- 适合场景：
- - 技术文档速读；
- 学习资料压缩；
- 视频/音频内容提炼；
- 论文或报告初筛。
- > 它最适合做"第一遍筛选"，帮你判断一份资料值不值得继续深读。
-

#### 四、效率办公类 Skills：Google、Notion、Obsidian


##### 9. GOG：Google Workspace 自动化
- GOG 是面向 Google Workspace 的自动化工具，支持：
- - Gmail；
- Google Drive；
- Google Docs 等服务。
- 典型用法：
- - 让 Claude 帮你发邮件；
- 整理 Google Drive 文件；
- 批量处理 Google Docs；
- 对文档内容进行总结、改写或归档；
- 自动生成邮件草稿。
- 适合对象：
- - 深度依赖 Google Workspace 的个人或团队；
- 需要批量处理邮件和文档的人；
- 经常在 Drive 中管理项目资料的团队。
- 如果主要使用飞书、钉钉等国内办公套件，这个 Skill 的优先级就不高。
- > 工具选择要看实际办公环境。Google 全家桶重度用户会觉得很香，不用 Google 的人基本可以跳过。
-

##### 10. Notion Skill
- Notion Skill 让 Claude 可以直接读写 Notion 页面和数据库。
- 它可以用于：
- - 整理笔记；
- 更新项目进度；
- 查询知识库；
- 生成周报；
- 管理任务记录；
- 把对话结论沉淀到 Notion。
- 一个典型玩法是周报自动化：
- ```text
  每周五让 Claude 读取本周任务记录，
  整理成 Notion 里的周报模板。
  ```
- 如果配合 Self Improving Agent 使用，还可以把 Claude 每次学到的项目经验、错误记录、个人偏好同步进 Notion 知识库。
> Notion Skill 的核心价值是把"对话中的知识"沉淀成"可复用的知识库"。

##### 11. Obsidian Skill
- Obsidian Skill 面向使用 Obsidian 管理本地笔记的用户。
- 它支持：
- - 在 Obsidian 库中搜索笔记；
- 创建笔记；
- 修改笔记；
- 处理双向链接；
- 将聊天结论保存到本地知识库。
- 典型场景：
- ```text
  和 Claude 讨论完一个技术方案后，
  直接让它把结论保存进 Obsidian，
  并补上相关双链。
  ```
- 适合对象：
- - 使用 Obsidian 做个人知识管理的人；
- 重视本地文件和 Markdown 笔记的人；
- 有双链笔记习惯的用户。
- 不使用 Obsidian 的人可以直接跳过。
> Obsidian Skill 适合"本地知识库派"，它的重点不是办公协作，而是长期个人知识沉淀。

#### 五、安全与维护类 Skills：先防风险，再谈效率


##### 12. Skill Waiter / Skill Better
- 字幕中出现了 "Skill Waiter" 和 "Skill Better" 两种说法，实际名称建议以插件市场为准。这里按视频语境理解，它指的是用于检查 Skill 安全性的工具。
- 它应该是安装新 Skill 前优先使用的安全检查类 Skill。
- 它可以检查：
- - Skill 申请了哪些权限；
- 是否包含可疑代码；
- 是否存在已知风险；
- 是否有明显的安全隐患。
- 因为 Claude Hub 属于开放社区，任何人都可以上传 Skill，质量和安全性参差不齐。也有人研究过如何在 Skill 中隐藏恶意代码。
- 这个工具不能保证 100% 安全，但能挡掉很多明显风险。
- 推荐使用流程：
- ```text
  准备安装新 Skill
  → 先用安全检查 Skill 扫一遍
  → 查看权限和风险提示
  → 再决定是否安装
  ```
> 安全类 Skill 不提升"爽感"，但它是整个 Skills 生态的底座。尤其是浏览器、GitHub、办公套件这类高权限 Skill，更应该先检查再安装。

##### 13. Auto Update
- Auto Update 用来自动更新已经安装的 Skills。
- 适合长期运行 Agent 的用户，因为你不需要手动记住：
- - 哪些 Skill 有新版本；
- 哪些版本修了 Bug；
- 哪些工具需要升级。
- 它可以自动完成更新，降低维护成本。
- 但需要注意：自动更新意味着你默认信任这些 Skill 的维护者。如果某个 Skill 权限很大，比如能操作浏览器、GitHub、Google Drive、Notion 数据库，那么不建议完全无脑自动更新。
- 更稳妥的方式是：
- - 普通低权限 Skill 可以自动更新；
- 高权限 Skill 建议手动更新；
- 更新前查看变更日志；
- 重大版本更新前先观察社区反馈。
> 自动更新提高维护效率，但也扩大了供应链风险。权限越大的 Skill，越不该闭眼更新。

#### 六、蒸馏类 Skills：把人和知识体系变成 AI Skill


##### 14. Dot Skill
- Dot Skill 前身叫 College Skill，曾在五天内增长 7500 个 Star，后来改名为 Dot Skill，功能也扩展了不少。
- 它的核心思路是：
- ```text
  把一个人的对话记录、文档、邮件等资料喂进去，
  生成一个可以模拟这个人的 AI Skill。
  ```
- 目前支持的蒸馏类型包括：
- | 类型 | 功能 |
|---|---|
| Colleague | 复刻同事的工作能力、表达方式、性格风格 |
| Relationship | 支持导入前任、伴侣、父母等聊天记录 |
| Celebrity | 支持乔布斯、马斯克等公众人物类型的数据蒸馏 |
- 支持的数据来源包括：
- - 飞书；
- 钉钉；
- Slack；
- 微信聊天记录；
- PDF；
- 邮件；
- 技术文档；
- Code Review 记录。
- 启动方式：
- ```text
  /dot skill
  ```
- 视频中提到的测试方式是：将一位同事的技术文档和 Code Review 记录喂进去，再询问接口设计是否有问题。生成结果不仅能模仿对方的技术判断，还能复现对方常用措辞。
- 关键影响因素：
- - 数据越完整，蒸馏结果越像；
- 数据质量越高，回答越可靠；
- 如果输入资料片面，生成的人格或能力也会偏差。
> Dot Skill 的重点不是简单"角色扮演"，而是把一个人的工作判断、语言习惯和知识结构压缩成可调用的 AI 能力。

##### 15. 八字 Skill
- 八字 Skill 是一个基于 Claude Code 的八字排盘工具，偏兴趣向，但安装人数也不少。
- 它的功能是根据用户输入的出生年月日时，参照传统命理典籍进行分析，包括：
- - 四柱八字排盘；
- 日主强弱；
- 十神关系；
- 大运；
- 流年；
- 命理结构分析。
- 视频中提到，它参考了九本经典命理典籍，不是随便糊弄的娱乐小工具，而是按照传统命理学框架来跑。
- 适合场景：
- - 个人兴趣；
- 偶尔娱乐；
- 给家里长辈演示；
- 了解传统命理框架。
- 需要注意的是，准确与否见仁见智，不应当把它当成严肃决策依据。
> 这个 Skill 的实用性不在生产力层面，更像是 Claude Code 生态多样性的一个例子。

### 推荐安装顺序
- 如果刚开始使用 Claude Code Skills，不建议一次性装满。可以按照下面顺序逐步安装：


#### Step 1：先装安全检查类 Skill
- 优先安装：
- ```text
  Skill Waiter / Skill Better
  ```
- 作用是安装其他 Skill 前先检查权限和风险。

#### Step 2：安装 Self Improving Agent
- 让 Claude 先具备错误记忆和自我改进能力。
- ```text
  Self Improving Agent
  ```
- 这是长期收益最高的一类 Skill。

#### Step 3：根据工作场景补充核心工具
- 开发者优先：
- ```text
  GitHub Skill
  Super Powers 或 Wassa to93
  ```
- 信息获取优先：
- ```text
  Summarize
  Agent Browser
  Multi Search Engine
  ```
- 知识管理优先：
- ```text
  Notion Skill
  Obsidian Skill
  ```
- 办公自动化优先：
- ```text
  GOG
  ```

#### Step 4：最后再考虑兴趣类和高级玩法
- 例如：
- ```text
  Dot Skill
  八字 Skill
  Auto Update
  Proactive Agent
  ```
- 其中 Proactive Agent 和 Auto Update 都涉及自动化执行，需要对权限和任务范围有清晰控制后再使用。
